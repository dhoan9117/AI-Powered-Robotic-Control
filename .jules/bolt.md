## 2024-05-24 - MediaPipe Processing Performance Optimization
**Learning:** By default, when passing a numpy array (like an image from OpenCV) to MediaPipe's `process()` method, MediaPipe makes a copy of the image if it is writeable. This is because MediaPipe's C++ backend requires the image data to not be modified during processing. Setting `image.flags.writeable = False` before passing it to `process()` passes the array by reference, significantly improving performance (often 10-20% faster execution time), especially when processing multiple models like Pose and Hands simultaneously on high-resolution video streams.
**Action:** Always set `.flags.writeable = False` on the RGB image array before passing it to any MediaPipe solution's `process()` method. Afterwards, if the image needs to be modified (e.g., for drawing landmarks), set `.flags.writeable = True`.
